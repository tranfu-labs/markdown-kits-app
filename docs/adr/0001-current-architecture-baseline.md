# 0001. 采用 React/Vite 前端与 Express JSON 分享服务作为当前架构基线

## 状态
accepted

## 背景
项目目标是复刻并优化一个微信公众号 Markdown 排版工具。核心体验主要发生在浏览器：编辑、预览、主题内联、本地图片、复制到公众号；服务端只需要提供短链分享和管理列表。

## 决策
当前架构采用 React + Vite + TypeScript 构建前端应用，采用 Express 提供分享 API，分享数据先落在 JSON 文件中并通过 `ShareStore.mutate()` 串行化写入。生产环境由同一个 Express 服务托管 `dist` 并处理前端路由回退。

## 后果
- 前端迭代速度快，核心排版链路可在浏览器内完成。
- API 面保持很小，方便本地开发和后续部署。
- JSON 文件存储适合轻量使用，不适合高并发或多实例部署；如果以后要横向扩展，需要引入数据库并新增 ADR。
- 分享服务不是渲染服务，渲染规则仍集中在前端 `src/lib/render.ts`。
